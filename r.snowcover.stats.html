NAME

r.snowcover.stats - Compute maximum, mean and minimum snow cover from data provided by an ImageServer

KEYWORDS

raster, ImageServer, snow, Sentinel-3, DTM, zonal statistics

SYNOPSIS

r.snowcover.stats
r.snowcover.stats --help
r.snowcover.stats [-d] [image_server=string] [dtm_service=string] [snow_service=string] aoi=name [date_start=string] [date_end=string] [snow_bands=integer] [min_snow_percent=integer] [--help] [--verbose] [--quiet] [--ui]

Flags:

-d
Write temporary data to disk (default is to compute in memory) (comutation in memory is faster but may limit number of days that can be processed)
--help
Print usage summary
--verbose
Verbose module output
--quiet
Quiet module output
--ui
Force launching GUI dialog

Parameters:

image_server=string
URL to ArcGIS ImageServer Typically starts with "http://"
Default: http://gis3.nve.no/image/rest/services/ImageService
dtm_service=string
Name of the service of the ImageServer to query for DTM
Default: AuxDEM250
snow_service=string
Name of the service of the ImageServer to query for snow cover data time series
Default: S3_SLSTR_fsc_sa
aoi=name [required]
Path to an OGR-readable file with Area of Interest
date_start=string
First date (in ISO-format: YYYY-MM-DD) of the time series to compute snow cover for (defaults to yesterday)
date_end=string
Last date (in ISO-format: YYYY-MM-DD) of the time series to compute snow cover for (defaults to today)
snow_bands=integer
Altitude steps (bands) in meter to compute snow cover for
Default: 250
min_snow_percent=integer
Minimum snow percent pre pixel to consider covered by snow
Default: 20

DESCRIPTION

r.snowcover.stats connects to an ArcGIS ImageServer, downloads a time series of snow cover maps and a terrain model (DTM) and computes minimum, mean and maximum snow cover over altitude bands. This is mainly an example for how to deploy python scripts in actinia.
